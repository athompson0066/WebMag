# Publishing Widget Studio: Universal Style Guide

## Global Architecture: The Editorial-Functional Split
All publishing widgets generated by the crews (Mini Apps, Chatbots, Lead Gen Forms, Course Pages, etc.) MUST strictly adhere to this vertical 2-column split-screen architecture on desktop.

### 1. Left Column: Editorial Hero (Brand & Narrative)
- **Visuals**: 
    - Display the `featuredImageUrl` prominently.
    - If background, use a soft white/light-gray overlay for text contrast.
- **Typography**:
    - **Primary Headline**: Oversized, bold serif font.
    - **Subtitle**: Bold sans-serif or italic serif.
    - **Description**: High-readability body copy.
- **Theme**: Strictly black text on light backgrounds (`text-black`).
- **Role**: This column tells the *story* and sets the *context*.

### 2. Right Column: Interactive Lab (Functionality)
- **Visuals**: Clean, modern, distraction-free "white paper" aesthetic.
- **Content**: This is where the interactive widget lives:
    - **Chatbots**: The conversation window.
    - **Courses**: The curriculum and enrollment logic.
    - **Lead Gen**: The input form.
    - **Mini Apps**: The primary interactive tool.
- **Theme**: Light backgrounds with black text. Primary accent colors used for call-to-action buttons.
- **Role**: This column handles the *utility* and *interaction*.

## Interaction Standards
- **Vanilla JS**: Components must be self-contained and use vanilla JavaScript.
- **Global Handlers**: Hook into `window.sendMessage`, `window.sendQuickReply`, and `window.trackEnrollment` where appropriate.
- **Responsiveness**: Columns must stack vertically on mobile (Editorial Hero on top, Functional Lab below).

## Visual Rules
- **No Dark Mode**: Maintain a high-contrast, premium light-themed aesthetic for readability.
- **Whitespace**: Ensure generous padding (minimum 2rem) between editorial copy and interactive components.
- **Typography Consistency**: Mix bold serif headers with clean sans-serif/monospace functional labels.
